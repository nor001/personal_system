## 2. Horario operativo base

### Día base (lunes a viernes)

| Hora | Actividad |
|---|---|
| 05:30–06:00 | Activación: agua, higiene, luz natural, movilidad / caminata breve |
| 06:00–06:50 | Bloque profundo 1 |
| 06:50–07:00 | Pausa |
| 07:00–07:50 | Bloque profundo 2 |
| 07:50–08:00 | Cierre del bloque / siguiente paso |
| 08:00–08:30 | Desayuno |
| 08:30–13:00 | Trabajo |
| 13:00–14:00 | Almuerzo |
| 13:30–13:50 | Revisión ligera opcional / retrieval / lectura técnica (solo si no interfiere con comida y descanso) |
| 14:00–17:30 | Trabajo |
| 17:45–18:00 | Pre-entreno ligero |
| 18:00–19:00 | Ejercicio o bloque aplicado según el día |
| 19:05–19:25 | Cena |
| 19:25–20:00 | Consolidación ligera / output / speaking / cierre del día |
| 20:00–05:30 | Sueño |

### Reglas del horario
- **La mañana construye:** los bloques profundos van primero.
- **La noche consolida:** después de entrenar y cenar solo va trabajo cognitivo ligero.
- **Objetivo realista de aprendizaje:** ~2h profundas por la mañana + 20–35 min de consolidación nocturna. La ventana de mediodía acerca el total a 3h sin convertir el día en fricción crónica.
- **Cena post-entreno:** no cenar completo antes de entrenar; usar pre-entreno ligero y cena moderada después.
- **Si el sueño empeora:** mover el cierre cognitivo nocturno a 10–20 min máximo y priorizar relajación.

### Distribución semanal base

| Día | 06:00–07:50 | 18:00–19:00 | 19:25–20:00 |
|---|---|---|---|
| Lunes | Inglés | Fuerza | Output ligero inglés |
| Martes | Proyecto principal | Zona 2 | Retrieval / revisión proyecto |
| Miércoles | Inglés | Fuerza | Shadowing / speaking / corrección |
| Jueves | Proyecto principal | Zona 2 | Revisión diferida / síntesis |
| Viernes | Inglés o proyecto principal según prioridad semanal | Fuerza | Cierre semanal ligero |
| Sábado | Escritura / PKM / síntesis / bloque extra del proyecto principal | Zona 2 si faltan minutos | Dashboard / planificación |
| Domingo | Descanso real / revisión ligera opcional | Descanso real | Dashboard breve / conexión social |

**Objetivo que protege:** reserva cognitiva, adherencia y sueño.

**Principio:** el mejor bloque cognitivo va donde hay más energía y menos fricción; el ejercicio de tarde corta el ciclo de distracción nocturna sin sacrificar la mañana.

**Regla:** dos bloques profundos fijos por la mañana; ejercicio principal por la tarde; solo consolidación ligera por la noche.

**Base:** práctica distribuida, gestión circadiana y adherencia conductual.

## 3. Proyecto principal del ciclo

**Proyecto principal:** Reposicionamiento profesional — ABAP senior → ABAP Cloud / RAP / clean core / SAP BTP ABAP environment con capa de IA aplicada a SAP  
**Tipo:** skill monetizable / cambio de rol / plataforma + IA aplicada  
**Estado:** construcción  
**Ventana del proyecto:** 2026-03-30 → 2026-06-21

### Objetivo del ciclo
- construir la columna vertebral moderna: ABAP Cloud + RAP + clean core + extensibilidad side-by-side
- añadir una capa diferencial de IA aplicada a procesos SAP/ERP, sin dispersarse en IA general
- producir evidencia pública o reusable del reposicionamiento

### Resultado concreto esperado al cierre
- [ ] mapa técnico cerrado del stack objetivo
- [ ] 1 demo funcional o mini-proyecto en ABAP Cloud / RAP
- [ ] 1 caso de extensibilidad clean core o integración vía APIs / eventos
- [ ] 1 artefacto de portfolio: write-up técnico, README estructurado o demo documentada
- [ ] 1 baseline claro para la capa de IA aplicada a SAP (caso de uso + diseño o prototipo inicial)

### Subenfoques del proyecto principal
1. **Columna vertebral (70%)**
   - ABAP Cloud
   - RAP
   - CDS
   - clean core / extensibility
   - SAP BTP ABAP environment
   - APIs / eventos / side-by-side
2. **Capa diferencial (20%)**
   - Joule for Developers / ABAP AI como herramienta, no como identidad
   - automatización, asistentes o flujos inteligentes sobre procesos SAP
3. **Laboratorio controlado (10%)**
   - IA general fuera de SAP solo si no compite con el foco principal

### Horas profundas objetivo por semana
- **6 bloques / semana**
- **5–6 horas profundas / semana**

### Evidencia de avance válida
- [ ] output generado
- [ ] conocimiento evaluado
- [ ] entregable claro
- [ ] siguiente paso definido

### Regla anti-cramming del proyecto
Ningún tema crítico del proyecto se trabaja en una sola exposición larga. Todo contenido importante debe tener como mínimo:
1. adquisición
2. recuperación temprana
3. recuperación diferida

### Orden de módulos del ciclo
1. Fundamentos de ABAP Cloud
2. RAP end-to-end
3. CDS y modelado asociado
4. Clean core / extensibilidad
5. Integraciones, APIs y eventos
6. Capa IA aplicada a SAP

### Plantilla de revisiones diferidas del proyecto

| Tema / módulo | Adquisición | Rev. 1 | Rev. 2 | Rev. 3 | Estado |
|---|---|---|---|---|---|
| Fundamentos ABAP Cloud | 2026-03-30 | 2026-03-31 | 2026-04-06 | 2026-04-20 | En curso |
| RAP end-to-end | 2026-04-07 | 2026-04-08 | 2026-04-14 | 2026-04-28 | Pendiente |
| CDS y modelado | 2026-04-15 | 2026-04-16 | 2026-04-22 | 2026-05-06 | Pendiente |

**Objetivo que protege:** capital cognitivo y opcionalidad profesional.

**Principio:** apalancar una ventaja previa fuerte suele dar más compound return que reiniciar identidad profesional desde cero.

**Regla:** la columna vertebral moderna SAP va primero; la IA entra como multiplicador encima, no como distracción paralela.

**Base:** reposicionamiento sobre moat previo, modernización ABAP, clean core y especialización diferencial.
